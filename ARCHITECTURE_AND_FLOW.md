# Architecture and Flow

## Overview

Interview Assistant App is organized into three main layers:

- `utils/` handles text extraction and cleaning
- `models/` handles AI logic and recommendation logic
- `app.py` connects everything through the Streamlit interface

The goal of the architecture is to keep the application simple, modular, and easy to extend.

## Layer Responsibilities

### 1. Utility Layer

The `utils` folder contains helper modules that prepare raw input before it is sent to the AI models.

#### `utils/cv_parser.py`

Responsibilities:

- receives the uploaded PDF CV file
- uses `PyMuPDF` (`fitz`) to extract text from each page
- cleans the extracted content
- returns a single string containing the CV text

#### `utils/JD_parser.py`

Responsibilities:

- receives the raw pasted job description
- removes unnecessary spacing and formatting noise
- returns a cleaned text string ready for prompting

### 2. Model Layer

The `models` folder contains the logic responsible for question generation, answer evaluation, and resource recommendation.

#### `models/question_generator.py`

Responsibilities:

- receives `cv_text` and `jd_text`
- builds the OpenAI prompt
- asks the model to generate interview questions
- parses the JSON response
- returns a Python list of questions

#### `models/answer_evaluator.py`

Responsibilities:

- receives a list of question and answer pairs
- sends them to OpenAI for evaluation
- requests a structured JSON response
- returns:
  - an overall score
  - a score per answer
  - feedback per answer

#### `models/resource_recommender.py`

Responsibilities:

- receives the evaluation result
- detects weak areas based on low-scored answers
- retrieves matching resources from `resources.json`
- optionally asks OpenAI for additional recommendations
- merges local and AI-generated resources
- labels each recommendation by source

#### `models/resources.json`

Responsibilities:

- stores the local library of curated learning resources
- organizes resources by topic
- acts as the stable recommendation base for the app

### 3. Application Layer

#### `app.py`

Responsibilities:

- provides the user interface with Streamlit
- collects user inputs
- calls each module in the correct order
- stores generated questions and evaluation results in session state
- displays questions, scores, feedback, and resources

## End-to-End Flow

The application follows this sequence:

1. The user uploads a CV in PDF format.
2. `cv_parser.py` extracts and cleans the CV text.
3. The user pastes a job description.
4. `JD_parser.py` cleans the job description text.
5. `question_generator.py` generates interview questions from the CV and job description.
6. The user writes answers to the generated questions.
7. `answer_evaluator.py` evaluates the answers and returns structured feedback.
8. `resource_recommender.py` identifies weak areas and returns recommended resources.
9. `app.py` displays the full result in the interface.

### Output Stage

- generated interview questions
- answer scores
- answer feedback
- overall score
- recommended learning resources

## Session State Usage

The app uses `st.session_state` to preserve important values across Streamlit reruns.

Stored values include:

- generated questions
- evaluation results

This allows the app to keep the interview flow stable even when the interface refreshes after button clicks.

## Recommendation Logic

The recommendation system uses a hybrid approach:

- local resources from `resources.json`
- additional resources generated by OpenAI

Each recommended resource includes a `source` field:

- `local_library`
- `ai`

This makes the recommendation flow more transparent.
